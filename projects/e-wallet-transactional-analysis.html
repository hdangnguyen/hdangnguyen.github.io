E-Wallet Transactional Analysis

A data wrangling project processing over 1.3 million records, focusing on data cleaning and missing value imputation via business logic. The cleaned dataset was then configured to answer analytical requirements and resolve operational tickets.

Data Cleaning Missing Value Imputation Jupyter Notebooks

01. Business Challenges

Operational and Product departments constantly require data analysis and categorization to drive strategic decisions. These teams consolidate their requirements and submit requests to the Data department. As the Data Analyst, my responsibility is to resolve the following tickets:

Ticket Business Requirement
Ticket 1 Identify the top 3 products with the highest total volume.
Ticket 2 Are there any products managed by more than one team? (Identify anomalies).
Ticket 3 Which team has had the lowest volume since Q2/2023, and which category contributes the least within that team?
Ticket 4 Analyze the contribution ratio of different source IDs in refund transactions.
Ticket 5 5a. Define a transaction_type for each row, given the classification rules:
  • Bank Transfer Transaction: transType=2, merchant_id=1205
  • Withdraw Money Transaction: transType=2, merchant_id=2260
  • Top Up Money Transaction: transType=2, merchant_id=2270
  • Payment Transaction: transType=2, other merchant_id
  • Transfer Money Transaction: transType=8, merchant_id=2250
  • Split Bill Transaction: transType=8, other merchant_id
  • Invalid Transaction: Remained cases
5b. Of each transaction type (excluding invalid): find the number of transactions, volume, senders and receivers.

02. Dataset Overview

The analysis is conducted using datasets provided by the internal Data Engineering team. These datasets represent a relational snapshot of the E-wallet's operational activities, comprising the following three tables:

  • transactions.csv: The core fact table containing highly granular daily transfer activities and transaction statuses.
  • payment_report.csv: Monthly aggregated payment metrics detailing volumes across various products and payment groups.
  • product.csv: A dimension table categorizing products and mapping them to their respective managing teams.

Ticket 3: Lowest Performing Team (Q2/2023 onwards)

Find the team with the lowest performance (lowest volume) since Q2.2023. Find the category that contributes the least to that team.

Python (Pandas) payment_full 
# create quarter to filter
payment_full['report_month'] = pd.to_datetime(payment_full['report_month'])
payment_full['quarter']      = payment_full['report_month'].dt.quarter

since_q2 = payment_full[payment_full['quarter'] > 1]

# find the lowest team
lowest_team = (
    since_q2.groupby('team_own')['volume']
    .sum()
    .sort_values()
    .index[0]
)
print(f"The team has the lowest volume since Q2/2023: {lowest_team}")

# lowest category
lowest_category = (
    since_q2[since_q2['team_own'] == lowest_team]
    .groupby('category')['volume']
    .sum()
    .sort_values()
    .index[0]
)
print(f"Category has the lowest contribution in team {lowest_team}: {lowest_category}")

Ticket 5a: Transaction Classification

Define a transaction_type label for each row based on the mapping below:

transType merchant_id transaction_type
2 1205 Bank Transfer Transaction
2 2260 Withdraw Money Transaction
2 2270 Top Up Money Transaction
2 others Payment Transaction
8 2250 Transfer Money Transaction
8 others Split Bill Transaction
Remained cases Invalid Transaction
Python (Pandas) transactions 
trans['transaction_type'] = 'Invalid Transaction'
for index, row in trans.iterrows():
  if row['transType'] == 2:
    if row['merchant_id'] == 1205:
      trans.loc[index, 'transaction_type'] = 'Bank Transfer Transaction'
    elif row['merchant_id'] == 2260:
      trans.loc[index, 'transaction_type'] = 'Withdraw Money Transaction'
    elif row['merchant_id'] == 2270:
      trans.loc[index, 'transaction_type'] = 'Top Up Money Transaction'
    else:
      trans.loc[index, 'transaction_type'] = 'Payment Transaction'
  elif row['transType'] == 8:
    if row['merchant_id'] == 2250:
      trans.loc[index, 'transaction_type'] = 'Transfer Money Transaction'
    else:
      trans.loc[index, 'transaction_type'] = 'Split Bill Transaction'

print(trans['transaction_type'].value_counts().to_string())

Ticket 5b: Transaction Summary

Of each transaction type (excluding invalid transactions): find the number of transactions, volume, senders and receivers.

Python (Pandas) transactions 
valid = trans[trans['transaction_type'] != 'Invalid Transaction']

summary = valid.groupby('transaction_type').agg(
    total_transaction   = ('transaction_id', 'count'),
    total_volume        = ('volume', 'sum'),
    number_of_senders   = ('sender_id', 'nunique'),
    number_of_receivers = ('receiver_id', 'nunique')
).sort_values('total_transaction', ascending=False)

print(summary.to_string())
